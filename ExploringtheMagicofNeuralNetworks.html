Problem Statement: Consider you have this dataset


Dataset

For instance, if you enter a value of -10, your output will be 14, if you enter a value of 0, your output will be 32, and so on. I want to ask you a question while you look carefully at these numbers.


Predict value 30

Can you predict what the output value will be if you use 30 as the input value? Before we show the solution, please take some time to find it.


Found the answer

You are correct, if you found the answer 86. Now I explain how this problem might be solved using both traditional programming and machine learning. Well, if you carefully look at the input and outputs, you will see that they are connected by the temperature conversion equation.


Celsius to Farenheit

F = C x 1.8 + 32

F = Fahrenheit C = Celsius


F equals C times 1.8 plus 32. Where the outputs refer to degree in Fahrenheit and are represented here by the letter F, and the inputs refer to degree in Celsius and are represented here by a letter C. In other words, the inputs represent a temperature value in degree Celsius, and the outputs display the corresponding temperature in degree Fahrenheit.


For example, an input value of zero refers to a temperature of zero degree Celsius, which corresponds to 32 degree Fahrenheit.


F = 0 x 1.8 + 32


we get 32 by multiplying zero with 1.8 and adding 32 like we see here in this equation. Similarly, an input value of 10 refers to a temperature of 10 degree Celsius which corresponds to 50 degree Fahrenheit. we get 50 multiplying 10 with 1.8 and adding 32 using the above equation.

What your brain did to find out the relationship between these specific inputs and outputs is exactly what machine learning algorithms performs. In other words, given a set of inputs and the corresponding outputs, machine learning algorithms will find the hidden patterns and relationship between the inputs to the outputs.


Traditional Programming vs Machine Learning

Traditional Approach


In traditional software development, the input and algorithm are known, and you write a function to produce an output. On the other hand, with machine learning, both the input and the output are known, but the algorithm that generates the output given the input is unknown, So the algorithm in machine learning needs to be learned.

If you wanted to create a computer program that could convert between degrees Celsius and degrees Fahrenheit using the formula F equals C times 1.8 plus 32, you could use the following formula: The use of a function in any programming language makes it simple to implement this relationship when using traditional software development. This function will look something like this in Python, for instance.


                def celsius_to_fahrenheit_convetor(C):
                  F = C * 1.8 + 32
                  return F

Here the function takes the input values C. Then computes the output value F using the explicit algorithm that you wrote down and finally returns the output value F.


Machine Learning Approach


On the other hand, the machine learning approach only includes the input and output; it does not include the algorithm. So it looks like this.

In the realm of machine learning, neural networks (NNs) serve as a vital foundation for solving a diverse range of problems. These networks are not merely single algorithms but rather a versatile class of algorithms that underpin the essence of modern machine learning. They excel in the art of discovering intricate patterns and representations within data, making them invaluable for various applications, from classification and regression to the recognition of complex patterns.

For the specific use case of converting Celsius to Fahrenheit, a single-layer neural network can adeptly capture the linear relationship between temperature values in two different scales. This simplified neural network architecture comprises just two layers: an input layer and an output layer. The input layer takes the Celsius temperature as input, and through the network's computational prowess, it elegantly transforms it into its Fahrenheit counterpart. This streamlined design showcases the efficiency of neural networks in handling straightforward yet essential tasks, and while it may be a minimalistic illustration of their capabilities, it exemplifies the adaptability and power inherent in these remarkable tools of modern machine learning.


Coding Part


Python and TensorFlow were used to complete this example.

Importing Libraries

                import tensorflow as tf
                import numpy as np
                import matplotlib.pyplot as plt

Creating Training Data

                celsius = np.array([-30, -20, -10, 0, 10, 20, 30, 40], dtype=float)
                fahrenheit = np.array([-22, -4, 14, 32, 50, 68, 86, 104], dtype=float)

Define the Model

  • input_shape=[1] — This specifies that the input to this layer is a single value.
  • units=1 — This specifies the number of neurons in the layer. The number of neurons defines how many internal variables the layer has to try to learn how to solve the problem.
                  #Build a layer 
                  model = tf.keras.Sequential([
                  tf.keras.layers.Dense(units=1, input_shape=[1])])

Compile the Model

  • Loss function — A way of measuring how far off predictions are from the desired outcome. (The measured difference is called the "loss".)
  • Optimizer function — A way of adjusting internal values in order to reduce the loss The loss function (mean squared error)and the optimizer (Adam) used here are standard for simple models like this one, but many others are available.
                  model.compile(loss='mean_squared_error',optimizer=tf.keras.optimizers.Adam(0.1))

Train the Model

The first argument is the inputs, the second argument is the desired outputs. The epochs argument specifies how many times this cycle should be run

                  history = model.fit(celsius, fahrenheit, epochs=1000)
                  print('done')
Model Training

Visualize the Model

The fit method returns a history object. We can use this object to plot how the loss of our model goes down after each training epoch.

                  plt.plot(history.history['loss'])

                  plt.ylabel("Loss Magnitude")
                  plt.title("Celsius to Fahrenheit")

It excels when the algorithm needs to be learned rather than explicitly defined.
